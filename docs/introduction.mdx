Skillscript is a small declarative language for authoring **durable, auditable, reusable agent skills** — routines an agent captures once and runs many times, instead of re-deriving the same workflow on every session.

A *skill* is a compiled, lint-checked, version-stamped procedure: declarative frontmatter (`# Skill:`, `# Vars:`, `# Triggers:`, `# Output:`) plus a small body of ops (`emit`, `notify`, `shell`, `file_write`, `$ <connector>.<tool>`, ...). Skills are substrate-neutral — the language doesn't bake LLM or storage identity into the source; adopters wire substrates through connectors and the same skill source runs against any conforming backend.

## At a glance

```
# Skill: hello
# Status: Approved
# Vars: WHO=world

Hello, ${WHO}!
```

That's a complete, valid skill. `Status: Approved` (with a stamped hash token) means the runtime will execute it; the body between frontmatter and the first target IS the output. Run it via `execute_skill({name: "hello"})` over MCP, or trigger it on a schedule via `# Triggers: cron: 0 9 * * *`.

For the full quickstart — install, author, dashboard, MCP wiring — see the [README on GitHub](https://github.com/sshwarts/skillscript#quickstart) or the [Adopter Playbook](/docs/adopter-playbook).

## Status

Skillscript is **pre-1.0**. The runtime surface freezes at the v1.0 cut; until then breaking changes are documented in the [CHANGELOG](https://github.com/sshwarts/skillscript/blob/main/CHANGELOG.md) but cheap (no external adopter install base yet).
